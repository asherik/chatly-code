# Product Vision

## Одно предложение

Chatly Code - локальный AI software engineer для desktop, который понимает проект, строит архитектурную карту, находит проблемы, создает задачи и безопасно меняет код под контролем разработчика.

## Что это не такое

- Не обычный чат с моделью.
- Не замена IntelliJ IDEA или VS Code.
- Не генератор кода "в пустоту".
- Не облачная IDE, куда обязательно надо отправлять весь проект.
- Не система, которая сама молча меняет файлы.

## Что это такое

Это локальная рабочая среда поверх существующего проекта:

- сканирует проект;
- объясняет архитектуру;
- строит C4;
- классифицирует проблемы;
- создает задачи;
- планирует изменения;
- правит код через controlled tools;
- запускает сборку и тесты;
- показывает diff;
- дает принять или откатить результат.

## Главная рыночная боль

Разработчики уже используют AI, но часто получают хаос:

- модель не понимает весь проект;
- код пишется без архитектурного контекста;
- контроллеры, сервисы и репозитории смешиваются;
- AI исправляет один баг и ломает соседний сценарий;
- нет четкого разделения bug/improvement/tech debt;
- нет безопасного workflow с branch, tests, rollback;
- сложно доверять изменениям в legacy-проекте.

Chatly Code должен закрыть эту боль через понятный процесс и сильный UX.

## Обещание продукта

Пользователь открывает старый проект и получает:

- карту проекта;
- C4-архитектуру;
- список проблем;
- список задач;
- AI-агента, который умеет работать пошагово;
- контроль над каждым рискованным действием;
- проверку тестами;
- diff-review перед принятием.

## Целевая аудитория

### Primary

- Backend-разработчики.
- Java/Spring-разработчики.
- Разработчики, которые поддерживают legacy.
- Solo-разработчики и indie hackers.
- Малые команды без выделенного архитектора.

### Secondary

- Team leads.
- Аутсорс-команды.
- Enterprise-команды с ограничениями на отправку кода в облако.
- Разработчики на Rust, JS, Python, Go, которым нужен локальный агент.

## Отличие от OpenHands

OpenHands дает агенту инструменты и runtime. Chatly Code должен взять этот фундаментальный подход, но усилить его в сторону desktop-продукта:

- local-first;
- architecture-first;
- task-first;
- safe-by-default;
- C4-first;
- project-understanding before coding;
- качественный desktop UX;
- постоянная память проекта.

## Ключевая формула

```text
OpenHands дает агенту руки.
Chatly Code должен дать агенту понимание проекта, архитектурную дисциплину и безопасный рабочий процесс.
```

## North Star

Пользователь должен чувствовать, что он стал техническим руководителем для AI-агента:

- сам не пишет рутину руками;
- выбирает задачи;
- утверждает план;
- контролирует качество;
- принимает готовые изменения.
